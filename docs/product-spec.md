## 1. 한 줄 정의

**연결된 실마리(Thread)를 따라 탐구하며 자신만의 지도(Map)를 만드는 아카이빙 플랫폼.**

## 2. 왜 만드는가

위키는 정답을 보여 주고, Letterboxd는 감상을 모아 주고, Obsidian은 개인 지식을 저장한다.
하지만 **"누가, 어떤 관점에서, 어떤 실마리들을 어떻게 이었는가"** 를 다루는 도구는 없다.

Silmaril은 다음을 한 곳에서 한다.

- 위키처럼 **공식화된 정보 구조**
- Letterboxd처럼 **개인 기록 / 컬렉션 문화**
- Obsidian처럼 **연결된 그래프**
- Instagram의 일부만 — **자유로운 기록 흐름**

## 3. 핵심 사용자

- **탐험자 (Explorer):** 좋아요 / 저장 / 컬렉션만으로 자신만의 지도를 모으는 사람.
- **기록자 (Recorder):** 본인의 관점을 기록하고, 가끔 공개하는 사람.
- **큐레이터 (Curator):** 관점을 다듬어 다른 사용자에게 보여 주는 사람. 일반 사용자 누구나 가능.
- **파트너 큐레이터 (Partner):** 명예 / 신뢰 배지를 받은 사용자. 권력이 아니라 노출 우선순위.

권한 차이가 아니라 **사용량과 의도의 차이**로 본다. 모든 사용자는 탐험 / 기록 / 검토 / 큐레이션이 가능하다.

## 4. 핵심 가치 / 원칙

| # | 원칙 |
|---|------|
| 1 | 실마리는 하나다. 같은 대상에 두 페이지를 두지 않는다. |
| 2 | 관점은 여러 개다. |
| 3 | 기록은 자유롭게 가능해야 한다. |
| 4 | 사용자의 기록 생성에 브레이크를 걸면 안 된다. |
| 5 | 공식화는 생성의 조건이 아니라 검증 이후의 상태다. |
| 6 | 모든 사용자는 탐험 / 기록 / 검토 / 큐레이션이 가능하다. |
| 7 | 파트너 큐레이터는 권력자가 아니라 명예 배지를 받은 사용자다. |
| 8 | 좋아요보다 저장과 연결이 중요하다. |
| 9 | 이 앱은 디자인 아카이브가 아니라 지식 탐험 플랫폼이다. |

## 5. 핵심 행동

| 행동 | 의미 | 시그널 강도 |
|------|------|------------|
| 좋아요 (`like`) | 가벼운 호감 | ⭐ |
| 저장 (`bookmark`) | "이거 내 지도에 두고 싶다" | ⭐⭐⭐ |
| 연결 (`connect`) | "이건 저것과 관련 있다" | ⭐⭐⭐⭐ |
| 컬렉션에 담기 (`collect`) | "내 큐레이션의 일부" | ⭐⭐⭐⭐ |
| 기록 작성 (`record`) | "내 관점이 있다" | ⭐⭐⭐⭐⭐ |
| 관점 작성 (`perspective`) | "공개해도 좋은 해석" | ⭐⭐⭐⭐⭐ |

좋아요는 정렬 기준으로 쓰지 않는다 (DECISIONS#D-006).

## 6. Thread 상태 모델

```
   create
      │
      ▼
  ┌────────┐    publish    ┌──────────┐  review   ┌──────────┐  curate  ┌──────────┐
  │ local  │ ─────────────►│community │ ────────► │ verified │ ───────► │ official │
  └────────┘               └──────────┘           └──────────┘          └──────────┘
      │                          │                      │                     │
      └────── merged_into ───────┴──────────────────────┴─────────────────────┘
                                       ▼
                                  ┌──────────┐
                                  │  merged  │
                                  └──────────┘

  (어느 상태에서나)  ─►  archived
```

- `local`: 본인만 본다. 본인의 지도/기록/컬렉션에는 즉시 사용 가능.
- `community`: 공개. 다른 사용자도 검색 / 연결 / 저장 가능. 검토 큐 진입.
- `verified`: 다수 검토 통과 (`>= N`명의 신뢰 큐레이터가 승인). 출처 1개 이상 필수.
- `official`: 실마릴 팀 / 파트너 큐레이터가 공식 인정.
- `merged`: 중복 실마리가 본체에 흡수됨. UI에서 항상 본체로 리다이렉트.
- `archived`: 부적절 / 비공개 처리.

## 7. 화면 (MVP)

### 7.1 Home `/`
- 오늘의 실마리
- 이어서 탐험하기 (마지막 본 실마리 → 인접 실마리)
- 추천 실마리 (저장 + 연결 시그널 기반)
- 큐레이터 활동 (최근 관점 / 공식화된 실마리)

### 7.2 Explore `/explore`
- 검색창 (이름 / 별칭)
- 추천 캐러셀
- 타입 탭: 인물 / 작품 / 사조 / 장소 / 사건 / 기관 / 기업 / 미디어 / 책 / 영화 / 음악

### 7.3 Create `/create`
하나의 진입점, 5개 하위 액션. **로그인만 되어 있으면 막지 않는다.**
- 실마리 생성 (Thread)
- 관점 작성 (Perspective)
- 개인 기록 작성 (Record)
- 컬렉션 생성 (Collection)
- 링크 저장 (Source) — URL 임베드 → 어느 실마리에 붙일지 묻기

### 7.4 Map `/map`
- 내 지도 (좋아요/저장/기록/컬렉션의 실마리를 노드로)
- 그래프 뷰 (React Flow)
- 필터: 좋아한 / 저장한 / 기록한 / 컬렉션
- 더블클릭 → Thread 상세

### 7.5 Profile `/profile`
- 사용자 정보 + 아바타
- 기록 수 / 컬렉션 수 / 발견한 실마리 수 / 연결한 실마리 수
- 파트너 배지

### 7.6 Thread 상세 `/threads/[id]`
- 헤더: 이름, 타입, 상태 배지, 별칭, 핵심 메타
- 탭: 관점 / 연결 / 기록 / 컬렉션 / 출처
- 작성/저장/연결/좋아요 액션 — 어느 탭에서도 가능

## 8. 비목표 (Non-goals, 적어도 MVP에서는)

- 실시간 협업 편집
- 댓글 스레드 (관점 자체가 그 역할)
- 위키 스타일의 풍부한 본문 편집기 — 본문은 마크다운 + 간단한 임베드만
- 추천 알고리즘의 ML — M5에서 검토
- 모바일 네이티브 앱

## 9. 성공 지표 (제품 관점, 트래픽 아님)

- **저장 / 연결 비율** — 좋아요 대비 저장 + 연결 시그널이 얼마나 발생하는가
- **재방문 시 지도가 자라는가** — `user_thread_activity` 의 신규 항목 수
- **공식화율** — `community` → `verified` → `official` 전환 비율
- **중복 발생률 / 병합률** — 같은 대상에 대한 `local` Thread가 몇 % `merged` 되는가
