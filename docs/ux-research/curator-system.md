# UX Research — Sprint 4: 큐레이터 시스템 (Curator System)

> 실마릴만의 핵심. "전문가가 **모더레이터가 되지 않고도** 어떻게 지위를 얻는가?"

## 연구 질문

- 평판/신뢰 시스템은 어떻게 작동하는가?
- 권력(게이트키핑) 없이 권위를 부여하는 방법은?
- 큐레이터 지위가 어떻게 발견 루프가 되는가?

## 추출한 UX 패턴

11. **평판 = 기여의 *가시성*, 권력 아님 (GitHub 기여 그래프, Genius IQ, Goodreads).** 배지는 누적된 양질 작업을 비출 뿐, 게이트키핑 권한을 주지 않는다.
12. **큐레이션 = 바이라인 관점 (Substack/Medium/Genius 주석).** 관점은 작성자 정체성을 달고 다닌다. 지위는 *좋은 관점*에 쌓이지, 편집 권한에 쌓이지 않는다.

## 강점

- **GitHub 모델:** 기여 그래프 = 권력 0, 신뢰 100. 아무도 막지 못하지만 누구나 인정한다.
- **Genius 모델:** 누구나 주석(=관점)을 달고, 좋은 주석이 IQ/배지로 보상됨 → "한 페이지, 여러 해석"의 실증.
- **Substack 모델:** 바이라인이 곧 신뢰. 팔로우가 발견 루프.

## 약점 / 안티패턴

- **모더레이터 권력화 (Wikipedia 편집 전쟁).** 권한이 곧 지위가 되면 텃세·게이트키핑·신규 진입 차단.
- 배지가 양(量) 기반이면 스팸 유발 (좋아요·게시물 수). → 질(저장·연결·인용) 기반이어야.
- 단일 "정답" 편집 모델은 관점 다양성을 죽인다 (문제 4의 반대편 함정).

## 실마릴 적용 (문제 매핑)

- **문제 4 (관점 많음 → 중복 피로):**
  - 같은 대상에 페이지를 여러 개 만들지 않는다. **하나의 실마리 위에 *바이라인 관점*을 레이어로 쌓는다** (Genius 주석 모델).
  - 중복 실마리는 새 페이지가 아니라 `merged` 로 흡수 (기존 D-005).
  - 관점은 "경쟁 페이지"가 아니라 "같은 페이지의 서로 다른 해석"으로 *나란히* 보여 피로 대신 깊이를 만든다.
- **파트너 큐레이터 = 명예 배지 (GitHub/Genius식), 권력 아님** (제품 철학과 정확히 일치).
  - 신뢰 점수는 저장·연결·인용·검토 통과 기반 (양이 아니라 질).
  - 공식화(`verified`/`official`)는 권한 행사가 아니라 *검증 통과의 상태*.
- 발견 루프: 좋은 큐레이터의 컬렉션/관점을 팔로우 → 새 실마리 유입 (Sprint 3과 연결).
