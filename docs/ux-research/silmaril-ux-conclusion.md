# Silmaril UX Research — 결론

> 4개 스프린트(탐험 / 그래프 / 컬렉션 / 큐레이터)의 증거를 **10개 패턴 + 8개 원칙**으로 수렴.
> 증거: Lazyweb 실앱 스크린샷 검색 + 지목 레퍼런스(Are.na, Obsidian, Letterboxd, Readwise, Wikipedia, Genius, GitHub, Substack) 분석.
> 슬로건: **Design with evidence, not vibes.**

## 한 문장 결론

실마릴은 위키(정보)도, 인스타(피드)도, Obsidian(그래프)도 아니다.
실마릴은 **"디자이너들의 지적 취향 지도"** 이며, 출시된 제품 중 **Are.na 에 가장 가깝다** — 단, 더 구조화(Wikipedia식 정보구조)되고 더 조용하다(Apple HIG).

## 실마릴이 푸는 5개 UX 문제

| # | 문제 | 이번 연구의 답 |
|---|------|----------------|
| 1 | 정보량 많음 → 안 읽음 | 상세를 글벽이 아니라 **연결+요약 카드**로 연다 |
| 2 | 탐험 → 길 잃음 | **이어보기 트레일 + 1-홉 이웃** 고정 |
| 3 | 좋아함 → 기록 안 함 | **저장을 1급 동사(한 탭)**, 좋아요는 부차 |
| 4 | 관점 많음 → 중복 피로 | **한 페이지 + 바이라인 관점 레이어** (Genius 모델) |
| 5 | 그래프 멋짐 → 안 씀 | 그래프는 **로컬·보조·리스트 기반**, 홈/내비 아님 |

## 추출한 UX 패턴 10

1. **한 대상 = 한 페이지. 관점은 바이라인 레이어** (Genius/Are.na). 중복 페이지 금지, `merged` 흡수. → 문제 4
2. **정보보다 연결을 먼저 보여준다.** 상세는 "이것이 연결된 것"으로 열린다 (Are.na connected). → 문제 1·2
3. **피드는 스캔 가능한 진입점(블록=탭).** 읽기 과제가 아니다 (Are.na/Pinterest). → 문제 1
4. **단 하나의 오늘의 앵커.** 선택 마비 제거 (NYT/Substack). → 문제 1
5. **돌아오면 "이어서 탐험" 트레일.** 콜드스타트 제거 (Readwise/Letterboxd). → 문제 2
6. **그래프는 로컬·보조·리스트 백업.** 실마리에서 진입, 1–2홉·라벨 엣지. 전역 그래프는 "내 지도 성취" 뷰로 격하 (Obsidian). → 문제 5
7. **저장은 한 탭, 절대 안 막음. 정리는 나중** (Pinterest/mymind). → 문제 3
8. **컬렉션 = 취향/정체성의 단위, 저장-into-컬렉션이 히어로 동사** (Are.na/Letterboxd/savee). → 문제 3·리텐션
9. **모으기보다 짓기.** "이건 네 지도의 ○○와 연결돼" 식 큐로 curiosity loop (Are.na). → 문제 2·3·리텐션
10. **평판 = 기여 가시성·바이라인, 권력 아님** (GitHub/Genius/Substack). 배지는 질(저장·연결·인용) 기반. → 철학·문제 4

## Silmaril UX Principles v1

1. **실마리는 하나, 관점은 여러 개.** 같은 대상에 페이지 둘을 만들지 않는다.
2. **연결을 정보보다 먼저 보여준다.** 글벽이 아니라 연결과 요약이 첫 화면.
3. **저장은 좋아요보다 중요하다.** 저장은 한 번의 탭, 정리는 나중. 좋아요는 휘발, 저장은 지도에 남는다.
4. **그래프는 목적이 아니라 탐험 도구다.** 로컬·보조·리스트 기반. 그래프 없이도 탐험이 끊기지 않아야 한다.
5. **사용자는 지식을 모으는 게 아니라 지도를 만든다.** 수집보다 연결. 탐험자는 기록 없이 저장·연결만으로 지도를 키운다.
6. **권위는 권력이 아니라 기여의 가시성이다.** 파트너 큐레이터는 명예 배지이지 게이트키퍼가 아니다.
7. **생성에 브레이크를 걸지 않는다.** `local` 즉시 사용, 중복은 나중에 `merged`.
8. **디자이너의 지적 취향 지도.** Are.na에 가깝게, 위키처럼 무겁지 않게, Apple HIG처럼 조용하게.

## 의도적으로 *하지 않을* 것 (anti-patterns)

- 무한 추천 피드(소셜미디어형) — 탐험이 아니라 소비를 만든다.
- 전역 그래프를 홈/1급 내비게이션으로.
- 좋아요 수·게시물 수 기반 랭킹/배지.
- 분류를 강제하는 무거운 폴더 — 저장 자체를 막는다.
- 단일 "정답" 편집 모델 — 관점 다양성을 죽인다.
