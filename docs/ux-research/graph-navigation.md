# UX Research — Sprint 2: 그래프 탐험 (Graph Navigation)

> 실마릴은 그래프가 핵심이라 믿어 왔다. 이 스프린트의 결론은 **"그래프는 핵심 도구이되 핵심 화면은 아니다."**

## 연구 질문

- 사람들은 왜 그래프 뷰를 결국 안 쓰게 되는가?
- 무엇이 그래프 내비게이션을 *유용하게* 만드는가?
- 모바일에서 그래프 탐험은 어떻게 작동해야 하는가?

## 증거 (Lazyweb 검색 결과 → 실제 앱)

| 쿼리 | 떠오른 실제 앱 (발췌) |
|------|----------------------|
| `knowledge graph visualization` | perplexity, **causaly**, **glean**, plasticity |
| `mind map` | **mymind**, notion, canva, anaplan |
| `relationship explorer` | **clay**, **affinity**, collibra, github |
| `node network map` | (대부분 무관 — 지도/내비 앱) |

**가장 중요한 음성 증거:** 순수 "노드 그래프 UI"를 메인으로 쓰는 출시 모바일 앱이 검색에 거의 안 떠오른다.
대신 떠오른 건 perplexity·glean·mymind·clay·affinity — **그래프를 *뒤에 숨기고* 리스트/카드/답변으로 표면화**한 제품들.
→ "모바일에서 그래프를 전면에 두면 안 쓰인다"는 가설을 데이터가 지지.

지목 레퍼런스: Obsidian · Roam · Kosmik · Fabric · Milanote · Are.na.

## 추출한 UX 패턴

5. **그래프는 보조/상세 뷰 (Obsidian local graph).** 전역 그래프가 아니라 "현재 노드의 이웃"으로. 홈이 아니라 *실마리에서* 진입.
6. **리스트 우선, 그래프는 보상.** 기본은 읽을 수 있는 연결 리스트(관계 라벨 포함). 그래프는 "줌아웃" 옵션. (안티패턴: 그래프를 랜딩으로.)
7. **제한된 로컬 그래프 (1–2홉, ≤~12 노드) + 라벨 있는 엣지.** 노드 수천 개의 전체 그래프는 장식이지 도구가 아니다 (Obsidian 전역 그래프가 안 쓰이는 이유).

## 강점

- 로컬 그래프는 "여기서 어디로?"라는 탐험 질문에 직접 답한다 (Obsidian backlinks).
- 관계에 *라벨*이 있으면(영향받음/소속/동시대) 그래프가 의미를 가진다 — 점-선이 아니라 문장이 된다.

## 약점 / 안티패턴

- **전역 그래프 = 멋지지만 무용 (문제 5의 정체).** 노드가 많아질수록 헤어볼이 되고, 모바일에선 줌/팬이 고통.
- 그래프를 1급 내비게이션으로 두면, 사용자는 결국 검색·리스트로 돌아간다.

## 실마릴 적용 (문제 매핑)

- **문제 5 (그래프 멋짐 → 안 씀):**
  - `/map`의 전역 그래프는 "내 지도"라는 *수집 성취* 뷰로 격하 (탐험 진입점 아님).
  - 진짜 탐험은 **실마리 상세의 "연결" 리스트**가 담당 (라벨 + 1-홉 이웃 카드).
  - 노드 더블탭 → 인접 실마리로 이동하는 **"한 홉씩 걷는" 모바일 인터랙션**만 지원. 전체 줌아웃은 부차.
- 모바일 우선: 그래프는 세로 스크롤 리스트로도 등가 표현 가능해야 한다(그래프 없이도 탐험이 끊기지 않게).
