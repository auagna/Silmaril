# Information Architecture — Silmaril

## 1. 라우팅 트리

```
/                              Home
/explore                       Explore
/explore?type=person           Explore + 타입 필터
/explore?q=...                 Explore + 검색
/create                        Create hub
/create/thread                 실마리 생성
/create/perspective?thread=…   관점 작성 (특정 실마리에)
/create/record                 개인 기록 작성
/create/collection             컬렉션 생성
/create/link                   링크 저장
/map                           지도 (그래프)
/map?filter=bookmarks          저장 필터
/map?filter=records            기록 필터
/map?filter=collections        컬렉션 필터
/map?filter=likes              좋아요 필터
/threads/[slug]                Thread 상세
/threads/[slug]/perspectives   관점 탭
/threads/[slug]/connections    연결 탭
/threads/[slug]/records        기록 탭
/threads/[slug]/collections    컬렉션 탭
/threads/[slug]/sources        출처 탭
/perspectives/[id]             단일 관점 상세 (공유용)
/collections/[id]              컬렉션 상세
/u/[handle]                    공개 프로필
/profile                       내 프로필
/profile/edit                  프로필 편집
/auth/sign-in                  로그인 (M2)
/auth/callback                 OAuth/매직링크 콜백
```

> MVP는 `/threads/[slug]` 의 탭을 URL로 분리하지 않고 클라이언트 탭으로 둔다 (스크롤/상태 보존).
> 위 트리는 향후 SEO를 위해 분리할 옵션을 열어 둔 형태.

## 2. 전역 네비게이션

상단(또는 좌측):

```
[ Silmaril ]   Home   Explore   Create   Map        ⌕  ◉ 프로필
```

- `Silmaril` 로고는 `/` 로 이동.
- 모바일은 하단 탭바: Home / Explore / Create(+) / Map / Profile.

## 4. 상태 / 권한 매트릭스

| 상태 | 작성자 본인 | 다른 사용자 | 큐레이터 | 검색 노출 |
|------|------------|------------|---------|-----------|
| `local` | R/W | ✗ | ✗ | ✗ |
| `community` | R/W | R | R/W (제안) | ◯ |
| `verified` | R | R | R/W | ◯ |
| `official` | R | R | R (변경은 admin) | ◯ (우선) |
| `merged` | R (리다이렉트) | R (리다이렉트) | merge/unmerge | ✗ |
| `archived` | R (본인) | ✗ | R (admin) | ✗ |

## 5. 상태 전이 트리거

- `local → community`: 작성자가 "공개" 버튼.
- `community → verified`: 누적 신뢰 점수 ≥ N **and** 출처 ≥ 1.
- `verified → official`: admin / 파트너 큐레이터 명시 액션.
- `any → merged`: 큐레이터가 본체 실마리를 선택.
- `any → archived`: admin 또는 작성자 본인.

## 6. URL slug 규칙

- 한글 그대로 허용. URL-encode 결과를 받아들임 (`/threads/디터-람스`).
- 충돌 시 `-2`, `-3` ... suffix.
- 별칭은 `aliases[]` 로 보관, slug 자체는 한 개.

## 7. 빈 상태 UX

| 화면 | 빈 상태 메시지 |
|------|----------------|
| Home — 이어서 탐험 | "최근 본 실마리가 없어요. **오늘의 실마리**부터 시작해 보세요." |
| Map | "아직 지도에 실마리가 없어요. **Explore에서 첫 실마리를 저장**해 보세요." |
| Profile — 기록 | "첫 기록을 남겨 보세요. 형식은 자유롭게." |
| Thread — 관점 | "이 실마리에 대한 첫 관점을 작성해 보세요." |

> 빈 상태에서도 다음 행동을 한 번의 탭으로 유도. 막다른 골목 금지.
