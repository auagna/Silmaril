# CLAUDE.md — Silmaril 공유 에이전트 프로토콜

이 파일은 Silmaril에서 일하는 **모든 에이전트의 공동 운영 규칙**이다.
Claude Code는 이 파일을 자동으로 읽고, Codex는 [AGENTS.md](./AGENTS.md)를 통해 이 파일로 안내된다.
**두 도구는 이 프로토콜을 동일하게 따른다.** (사람이 직접 작업할 때도 동일.)

---

## 0. 교대 모델 (가장 중요)

- 작업은 **Claude Code ↔ Codex 가 번갈아** 진행한다.
- 한쪽 이용량이 소진되면 다른 쪽이 **같은 지점에서 이어받는다.**
- 이어받는 지점의 단일 진실 공급원(SSOT)은 단 두 개다:
  1. **`HANDOFF.md` → "현재 상태"** (지금 어디까지 됐고, 진행 중이던 건 무엇인가)
  2. **`TASKS.md` → "## Now"** (다음에 할 일, 위에서부터 하나씩)
- 교대는 **git** 으로 이뤄진다. 세션을 끝낼 때 **반드시 커밋**하고, 새 세션은 **git 최신 + HANDOFF.md** 를 기준으로 시작한다.
- 원격: https://github.com/auagna/Silmaril — 세션 종료 시 push, 시작 시 pull 권장.

---

## 1. 세션 시작 시 (반드시, 순서대로)

아래 파일을 이 순서로 읽는다:

1. `README.md`
2. `DECISIONS.md`
3. `TASKS.md`
4. `HANDOFF.md`
5. `docs/product-spec.md`
6. `docs/data-model.md`
7. `docs/information-architecture.md`
8. `docs/supabase-schema.md`

그 후 현재 코드 상태를 파악한다. (어떤 라우트/컴포넌트가 있고, 더미인지 실데이터인지.)

## 2. 무엇을 할지 고른다

- `TASKS.md` 의 **`## Now`** 에서 **체크되지 않은 첫 번째 항목 하나만** 진행한다.
- `Now` 가 비어 있으면 `## Next` 의 첫 항목 하나를 `Now` 로 올린 뒤 진행한다.
- **한 번에 여러 기능을 추가하지 않는다.** 하나를 끝내고 커밋하고 멈춘다.

## 3. 작업 규칙

- **기존 구조를 임의로 바꾸지 않는다.** 폴더 구조, 파일 위치, 네이밍을 마음대로 재배치하지 말 것. 확장은 OK, 재배치는 멈추고 `DECISIONS.md` 에 근거를 남긴 뒤에만.
- 모든 주요 타입은 **TypeScript** 로 (`src/types/`).
- **Supabase 클라이언트는 `src/lib/supabase.ts`** 에 둔다. (SSR 헬퍼를 추가할 땐 같은 lib 폴더 안에서 확장.)
- UI 컴포넌트는 **재사용 가능하게** `src/components/` 또는 도메인별 `src/features/*` 로 분리.
- **API Key 를 코드에 직접 넣지 않는다.** `.env.example` 만 갱신, 실제 키는 `.env.local` (gitignore됨).
- 모호한 부분은 **`DECISIONS.md` 의 결정이 우선**한다. 결정에 없으면 새 결정을 `DECISIONS.md` 에 추가하고 진행.

## 4. 세션 종료 시 — 교대 핸드오프 체크리스트

다음 에이전트가 끊김 없이 이어받도록, **멈추기 전에 모두 수행**한다:

- [ ] 완료한 작업을 `TASKS.md` 에서 `[x]` 로 체크.
- [ ] `HANDOFF.md` 의 "현재 상태" 갱신:
      - 무엇을 끝냈는가
      - **진행 중이던 작업이 있으면** 정확히 어디까지(파일 + 위치 + 다음 한 걸음)
      - 다음에 할 일 (= `TASKS.md ## Now` 의 첫 항목)
      - 알려진 위험 / 막힌 지점
- [ ] **변경 파일 목록 / 실행 방법 / 테스트 방법** 을 응답 마지막에 정리.
- [ ] 레포를 **빌드 가능한 상태**로 남긴다. 불가하면 그 사실을 HANDOFF에 명시.
- [ ] **커밋**. (가능하면 push.) 커밋 메시지에 어느 `Now` 항목을 끝냈는지 적는다.

## 5. 앱 철학 (모든 결정의 상위 기준)

> **실마리는 하나, 관점은 여러 개, 기록은 자유롭게. 공식화는 검증 이후.**

- 같은 대상에 페이지를 두 개 만들지 않는다. 중복은 `merged` 로 흡수.
- 하나의 실마리에 여러 관점(Perspective)이 붙는다.
- 사용자의 **기록 생성 흐름에 브레이크를 걸지 않는다.** `local` 상태도 본인 지도/기록에는 즉시 반영.
- **좋아요보다 저장·연결·컬렉션**이 1등급 시그널. 정렬을 좋아요로 하지 않는다.

## 6. 충돌 / 모호함

- 지시가 충돌하면: **DECISIONS.md > 이 파일(CLAUDE.md) > 그 외**.
- 결정이 필요한데 근거가 없으면, 합리적 기본값으로 진행하되 `DECISIONS.md` 에 한 줄 남긴다.
